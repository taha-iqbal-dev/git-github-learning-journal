# Module 01

## Introduction to Version Control

Version control is a system that save history of changes in a file overtime so that these can be recalled later

### Benefits

- Trackes file history
- Allows collaboration
- Prevent accidental loss of work
- Makes recovery easier

## Keeping historical copies

Without version control people saved files like

report_v1.docx
report_v2.docx
report_final.docx
report_final_final.docx

This quickly becomes difficult to understand
VCS makes all this easier to manage and understand

## Diff

Diff compares tw versions of the file and show the difference

Benefits
- Understand changes
- Review modification
- Track updates efficiently

## Patch

Patch apply changes generated by Diff

Workflow
1. Compare files via diff
2. Generate a patch file
3. Apply patch to another file

## What is Git?

Git is a distributed version control system.

Features:

- Fast and Reliable
- Support collaborations
- Maintain complete project history

## Basic Git Workflow

Working Directory --> Staging Area --> Repository (Commit History)

Commands:

git add
git commit
git status
git log

## Commit Messages

Good commit messages describe what changed.

Examples:

"Add project README"

"Create version control notes"

"Add diff demonstration"
